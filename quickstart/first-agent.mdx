title: Build your first agent
description: Step-by-step guide from API key to your first browser test call.
---

Follow each step in order. At the end you'll talk to your voice agent in the browser.

**Time:** ~15 minutes

---

## What you need before step 1

| Item | Where to get it | Used in |
| --- | --- | --- |
| **OneInbox API key** | OneInbox dashboard → **API Keys** → **Create API key** | Every step |
| **OpenAI API key** | [platform.openai.com](https://platform.openai.com) → API keys → Create | Step 2 |
| **Terminal** | Mac: Terminal app. Windows: PowerShell or WSL | Every step |

You do **not** need: Twilio, a phone number, ElevenLabs, or a credit card on OneInbox for this guide.

<Info>
Dashboard URL is provided during onboarding. Log in there to create and manage API keys.
</Info>

<Tip>
**How to get an OpenAI key:** Log into OpenAI → click your profile → **API keys** → **Create new secret key** → copy it (starts with `sk-`). You won't see it again.
</Tip>

---

## How to use this guide

Every command has **placeholders** in angle brackets. Replace them with your real values before running:

| Placeholder | Replace with |
| --- | --- |
| `<api_key>` | Your OneInbox key from the dashboard — starts with `oi_live_` |
| `<your_openai_api_key>` | Your OpenAI key — starts with `sk-` |
| `<llm_id>` | ID from step 3 |
| `<agent_id>` | ID from step 4 |
| `<call_id>` | ID from step 5 |

**Keep a notepad** (or use this template) and fill in values as you go:

```
api_key: 
llm_id: 
agent_id: 
call_id: 
meet_url: 
```

<Steps>

<Step title="1. Get your OneInbox API key">

### What you're doing

You're copying your **API key** — the credential used in every curl command in this guide.

### How

1. Log in to the **OneInbox dashboard** (URL from your onboarding email)
2. Go to **API Keys**
3. Click **Create API key**, enter a name (e.g. "My First Key"), and copy the key

<Warning>
**Copy the key now.** OneInbox shows the full value **only once**. If you lose it, create a new key in the dashboard and revoke the old one.
</Warning>

<Check>
**Save:** `api_key` (starts with `oi_live_...`) → use `Authorization: Bearer <api_key>` in **every step from here on**.
</Check>

More detail → [Authentication](/concepts/authentication)

</Step>

<Step title="2. Connect OpenAI (create an Integration)">

### What you're doing

You're giving OneInbox permission to use **your** OpenAI account when your agent needs to think.

### Why?

OneInbox doesn't use a shared OpenAI key. You bring your own. We encrypt it and store it — the raw key is **never** returned in API responses after this step.

### What is an Integration?

An **Integration** is a stored third-party API key. In the API it's called `/v1/credentials`, but we say "Integration" in docs because that's what it represents.

---

**Replace:**
- `<api_key>` → your OneInbox key from step 1
- `<your_openai_api_key>` → your OpenAI key (`sk-...`)

```bash
curl -X POST http://13.207.23.185:8000/v1/credentials \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "OpenAI",
    "provider": "openai",
    "api_key": "<your_openai_api_key>"
  }'
```

**What success looks like** (HTTP `201`):

<ResponseExample>

```json
{
  "id": "cred_openai_abc123",
  "name": "OpenAI",
  "provider": "openai",
  "created_at": "2026-01-15T10:01:00Z"
}
```

</ResponseExample>

Notice there is **no** OpenAI key in the response — that's intentional.

<Check>
**Save (optional):** `id` as `credential_id`. For this basic guide, step 3 uses `"provider": "openai"` and OneInbox finds your stored key automatically.
</Check>

**Common errors:**

| Error | Meaning | Fix |
| --- | --- | --- |
| `401` | Missing or wrong API key | Use full `oi_live_...` key from dashboard |
| `400` | Invalid provider or key | Check `"provider": "openai"` and your `sk-` key |

</Step>

<Step title="3. Create an LLM model (the brain)">

### What you're doing

You're defining **how your agent thinks** — which GPT model to use and what instructions it follows.

### Why?

The **Agent** (step 4) doesn't contain the prompt directly. It points to an **LLM model** via `llm_id`. This lets you reuse the same brain across agents or update prompts without rebuilding the whole agent.

### Fields explained

| Field | What it means | Example in this guide |
| --- | --- | --- |
| `name` | Label for your dashboard | `"My First Model"` |
| `provider` | Which AI company | `"openai"` |
| `model` | Which model version | `"gpt-4o-mini"` (fast + cheap for testing) |
| `system_prompt` | Instructions the agent always follows | Personality, tone, rules |
| `temperature` | Creativity (0 = strict, 1 = creative) | `0.7` |
| `tool_ids` | Actions the agent can take | `[]` empty for now |
| `knowledge_base_ids` | Docs to search | `[]` empty for now |

Feel free to change the `system_prompt` to match your use case — e.g. "You are a pizza shop order taker..."

---

**Replace** `<api_key>` with your OneInbox key.

```bash
curl -X POST http://13.207.23.185:8000/v1/models \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My First Model",
    "provider": "openai",
    "model": "gpt-4o-mini",
    "system_prompt": "You are Alex, a friendly assistant. Keep every response under two sentences. Be warm and direct.",
    "temperature": 0.7,
    "max_tokens": 4000,
    "tool_ids": [],
    "knowledge_base_ids": []
  }'
```

**What success looks like** (HTTP `201`):

<ResponseExample>

```json
{
  "id": "model_abc123",
  "name": "My First Model",
  "provider": "openai",
  "model": "gpt-4o-mini",
  "system_prompt": "You are Alex, a friendly assistant...",
  "temperature": 0.7,
  "created_at": "2026-01-15T10:02:00Z"
}
```

</ResponseExample>

<Check>
**Save:** the `"id"` field as **`llm_id`**. You **must** paste this into step 4.
</Check>

**Common errors:**

| Error | Meaning | Fix |
| --- | --- | --- |
| `401` | Bad API key | Re-check `api_key` from step 1 |
| `402` / OpenAI error | OpenAI billing or key issue | Check OpenAI dashboard has credits |

</Step>

<Step title="4. Create an Agent (listen + think + speak)">

### What you're doing

You're assembling the full **voice agent** — the thing that actually takes calls.

An agent combines:

| Part | Config field | What it does in this guide |
| --- | --- | --- |
| **Brain** | `llm_id` | Links to step 3 |
| **Ears (STT)** | `transcriber` | Deepgram listens to your speech |
| **Voice (TTS)** | `tts` | Deepgram speaks replies (voice: asteria) |
| **Behavior** | `first_message`, timeouts | What Alex says first, when to hang up |

### Behavior fields explained

| Field | What it does |
| --- | --- |
| `first_message` | First thing the agent says when the call connects |
| `end_call_phrases` | If the **user** says these words, the call ends |
| `silence_timeout_seconds` | Hang up after this many seconds of silence |
| `max_duration_seconds` | Hard limit on call length (600 = 10 minutes) |

---

**Replace:**
- `<api_key>` → your OneInbox key
- `<llm_id>` → ID from step 3

```bash
curl -X POST http://13.207.23.185:8000/v1/agents \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alex",
    "llm_id": "<llm_id>",
    "transcriber": {
      "provider": "deepgram",
      "model": "nova-3",
      "language": "en"
    },
    "tts": {
      "provider": "deepgram",
      "voice_id": "asteria",
      "speed": 1.0,
      "stability": 0.5
    },
    "first_message": "Hi! I am Alex. How can I help you today?",
    "end_call_phrases": ["goodbye", "bye", "that is all"],
    "silence_timeout_seconds": 10,
    "max_duration_seconds": 600
  }'
```

**What success looks like** (HTTP `201`):

<ResponseExample>

```json
{
  "id": "agent_abc123",
  "name": "Alex",
  "llm_id": "model_abc123",
  "transcriber": { "provider": "deepgram", "model": "nova-3", "language": "en" },
  "tts": { "provider": "deepgram", "voice_id": "asteria" },
  "first_message": "Hi! I am Alex. How can I help you today?",
  "created_at": "2026-01-15T10:03:00Z"
}
```

</ResponseExample>

<Check>
**Save:** the `"id"` field as **`agent_id`**. You need it for step 5.
</Check>

**Common errors:**

| Error | Meaning | Fix |
| --- | --- | --- |
| `404` on llm_id | Wrong or missing LLM ID | Re-copy `id` from step 3 |
| `400` | Invalid transcriber/tts config | Use the exact JSON above for first test |

</Step>

<Step title="5. Start a browser test call">

### What you're doing

You're starting a **live conversation** with your agent. OneInbox returns a link — you open it in Chrome or Safari, allow your microphone, and talk.

### Is this production?

**No.** Browser test calls are for **development and demos**. For real phone calls to customers, see [Phone calls](/quickstart/phone-calls) after this guide.

### What you'll experience

1. You run the curl below
2. You get a `participant_token` and `server_url`
3. You open a meet link in your browser
4. Alex says the `first_message` from step 4
5. You speak — Alex replies
6. Say **"goodbye"** to end (matches `end_call_phrases`)

---

**Replace:**
- `<api_key>` → your OneInbox key
- `<agent_id>` → ID from step 4

```bash
curl -X POST http://13.207.23.185:8000/v1/calls/web \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "<agent_id>",
    "variables": { "customer_name": "Alex" }
  }'
```

**What success looks like** (HTTP `201`):

<ResponseExample>

```json
{
  "id": "call_abc123",
  "status": "queued",
  "server_url": "wss://livekit.example.com",
  "participant_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

</ResponseExample>

### Open the meet link

Build this URL by swapping in your values (keep the `wss://` prefix on `server_url` as-is):

```
https://meet.oneinbox.ai?token=<participant_token>&server=<server_url>
```

**Example** (yours will differ):

```
https://meet.oneinbox.ai?token=eyJhbGci...&server=wss://livekit.example.com
```

Paste the full URL into your browser address bar and press Enter.

<Tip>
**Browser tips:** Use Chrome or Safari. Click **Allow** when asked for microphone access. Use headphones to avoid echo.
</Tip>

<Check>
**Save:** `"id"` as **`call_id`** for the verify step below.
</Check>

**Common errors:**

| Error | Meaning | Fix |
| --- | --- | --- |
| `404` on agent_id | Wrong agent ID | Re-copy from step 4 |
| Meet page loads but no audio | Mic blocked | Check browser permissions |
| Agent doesn't speak | Wait 5–10 seconds | First connection can take a moment |

</Step>

</Steps>

## Verify your call worked

After you say goodbye and the call ends, fetch the call record:

**Replace** `<call_id>` and `<api_key>` with your values.

```bash
curl http://13.207.23.185:8000/v1/calls/<call_id> \
  -H "Authorization: Bearer <api_key>"
```

**What to look for:**

<ResponseExample>

```json
{
  "id": "call_abc123",
  "status": "completed",
  "agent_id": "agent_abc123",
  "transcript": [
    { "role": "assistant", "content": "Hi! I am Alex. How can I help you today?" },
    { "role": "user", "content": "What's the weather like?" },
    { "role": "assistant", "content": "I don't have weather data, but I'm happy to help with something else!" }
  ],
  "duration_seconds": 42
}
```

</ResponseExample>

If `status` is `"completed"` and you see a `transcript` array — **you're done.** You built a working voice agent.

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized on any step">
    Missing or wrong API key. Use `Authorization: Bearer <api_key>` with your full `oi_live_...` key from the dashboard. Create a new key if you lost it.
  </Accordion>
  <Accordion title="OpenAI errors during the call">
    Check your OpenAI account has billing enabled and the key in step 2 is valid. Test the key at platform.openai.com.
  </Accordion>
  <Accordion title="curl: command not found">
    Install [curl](https://curl.se/) or use the **Try It** buttons in the [API reference](/api-reference/integrations/create-integration) instead.
  </Accordion>
  <Accordion title="JSON parse error from curl">
    Make sure quotes in `-d '{ ... }'` are straight quotes, not curly “smart quotes” from Word or Notes.
  </Accordion>
  <Accordion title="Agent works in browser but I want phone calls">
    Complete [Phone calls](/quickstart/phone-calls) next — you'll add Twilio and dial real numbers.
  </Accordion>
</AccordionGroup>
