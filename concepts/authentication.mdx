## The two tokens

When you sign up, OneInbox gives you a **JWT** (`access_token`). When you create an API key, you get an **`api_key`**. They look different and do different jobs.

| | JWT (`access_token`) | API key (`api_key`) |
| --- | --- | --- |
| **Looks like** | Long string starting with `eyJ...` | Starts with `oi_live_...` |
| **How long it lasts** | Session / expires | Until you revoke it |
| **Use for** | Sign-up, login, **create/revoke API keys only** | **Everything else** |
| **Get it from** | `POST /v1/auth/signup` or `/login` | `POST /v1/api-keys` |

## Which header to send

Every authenticated request uses:

```
Authorization: Bearer <token>
```

But **which token** depends on the endpoint:

| Endpoint | Token to use |
| --- | --- |
| `POST /v1/auth/signup` | None |
| `POST /v1/auth/login` | None |
| `POST /v1/api-keys` | JWT |
| `DELETE /v1/api-keys/{id}` | JWT |
| **All other `/v1/*` endpoints** | **API key** |

```bash

  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \

```

Copy `api_key` immediately â€” shown once.

**3. Everything else** â€” use API key:

```bash

  -H "Authorization: Bearer <api_key>" \
  ...
```

## Common mistakes

### Using JWT after step 2

**Symptom:** `401 Unauthorized` when creating integrations, agents, or calls.

**Fix:** Use `api_key` (`oi_live_...`), not `access_token` (`eyJ...`).

### Lost your API key

You cannot retrieve it again. Either:

1. Log in â†’ get a new JWT â†’ create a new API key, or
2. Revoke the old key and create a fresh one

### No auth header at all

**Symptom:** `401` on protected endpoints.

**Fix:** Add `-H "Authorization: Bearer <api_key>"` to every request except sign-up and login.
