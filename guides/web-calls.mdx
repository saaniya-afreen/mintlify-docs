## Overview

A **web call** is a call session you create with the OneInbox API. It powers browser-based voice on your backend.

OneInbox returns `call_id`, `server_url`, and `participant_token` when you create a web call.

**Prerequisites:** a working `agent_id` from [Quickstart](/guides/quickstart).

---

## What you get back

| Field | Purpose |
| --- | --- |
| `call_id` | Identify the session — poll with `GET /v1/calls/<call_id>` |
| `server_url` | WebSocket URL (e.g. `wss://voice.oneinbox.ai`) — pass to your frontend |
| `participant_token` | Short-lived client token — pass to your frontend, not your API key |

---

<Steps>

<Step title="1. Create a web call">

```bash
curl -X POST https://api.oneinbox.ai/v1/calls/web \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "<agent_id>",
    "variables": { "customer_name": "Guest" }
  }'
```

<ResponseExample>

```json
{
  "id": "call_abc123",
  "server_url": "wss://voice.oneinbox.ai",
  "participant_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

</ResponseExample>

</Step>

<Step title="2. Pass per-call context (optional)">

Use `variables` to inject context into the agent prompt:

```json
{
  "agent_id": "<agent_id>",
  "variables": {
    "customer_name": "Sam",
    "order_id": "ORD-001"
  }
}
```

Reference in your system prompt: `The customer name is {{customer_name}}.`

</Step>

<Step title="3. Override voice or model per call (optional)">

Change TTS or LLM for a single call without updating the agent:

```bash
curl -X POST https://api.oneinbox.ai/v1/calls/web \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "<agent_id>",
    "overrides": {
      "tts": {
        "provider": "cartesia",
        "voice_id": "a0e99841-438c-4a64-b679-ae501e7d6091"
      },
      "first_message": "Hi! How can I help you today?"
    }
  }'
```

See [Start web call](/api-reference/calls/start-web-call) and [Voices](/guides/voices) for all override fields.

</Step>

<Step title="4. Verify the call">

```bash
curl https://api.oneinbox.ai/v1/calls/<call_id> \
  -H "Authorization: Bearer <api_key>"
```

Look for `"status": "completed"` and a `transcript` array when the conversation ends.

</Step>

</Steps>

---

## Security

Return only `server_url` and `participant_token` to your frontend — never your API key.

---

## Web call vs phone call

| | Web call | Phone call |
| --- | --- | --- |
| **Channel** | Browser / web app | PSTN via telephony carrier |
| **Phone number** | Not required | Required |
| **Telephony integration** | Not required | Required |
| **Guide** | This page | [Phone calls](/guides/phone-calls) |

---

## Troubleshooting

| Problem | Fix |
| --- | --- |
| `401` | Check `Authorization: Bearer <api_key>` |
| `404` on agent_id | Re-copy `agent_id` from [Quickstart](/guides/quickstart) step 2 |
| Tokens not working in browser | Create a fresh web call — tokens are short-lived |
